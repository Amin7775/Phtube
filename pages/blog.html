1. Discuss the scope of var, let, and const

In programming, scope is an area where variables stay. It defines where a variable can work or not.Their are 2 types of scope. Global and Local.


Var, let and const are 3 different types of variable declaration methods.They have their own scopes. among those 3, let and const are more similar to each other and more powerful than var. they are also preferred more than var


var scope:

Var is a old declaration type of js which was used before es6 .

The variables which are declared using var can be used anywhere within the function. it means that if there is any function inside that another function and var is declared on the inside function , we can still use it on the outer function


let and const scope:

Let and Const are new types of variable declarations. They were introduced in the ES6 Js update in 2015.

let and const are the block type scopes. They are only access-able when they are used inside the function they are declares.
For example, If they were declared inside a function and their is another function outside , they cannot be used on the outside function. As a result they are more secure to use. Because of this feature, let and const is more preferred than the var .


2. Tell us the use cases of null and undefined

Null and Undefined represents two different meanings of a value and they have different uses


Null :

Null is a type of value which shows up when when a variable or object has no value in it.Uses of it -
1. We can assign a null is a variable when we are declaring it. it means that the variable will be declared with no value in the start.
2. To check if any value exists inside the object or variable.


Undefined :

Undefined is shown when a value is absent from a variable or data is in uninitialized state. Its shows up when -
1. Function has no return.
2. Variable that is not initialized.
3. Property that does not exist.
4. Parameter that has not been passed.
